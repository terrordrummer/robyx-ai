## Documentation

Documentation-only release: brings `README.md`, `ORCHESTRATOR.md`, and `SCHEDULER.md` in sync with everything that shipped in the v0.11.x → v0.12.x line. No code changes, no schema changes, no new dependencies.

### What was updated

**`README.md`**

- **Tagline and intro** are now platform-agnostic: "Your AI staff, managed from chat" with explicit mention of Telegram, Discord, and Slack. Clone/Configure/Talk now says "each with its own chat topic/channel" instead of Telegram-specific framing.
- **Command Bridge** concept introduced in the "How It Works" and "Kael — The Orchestrator" sections. Kael no longer "lives in the Main channel" anywhere in the README — everywhere it now "lives on the Command Bridge". The ASCII diagram in "The Three Roles" is updated accordingly.
- **Coordination-first contract** documented in the Kael section: the Bridge is for fleet status, workspace creation, delegation, and meta-operations; project-specific work (R&D iterations, builds, deploys, feature implementation) goes through `[DELEGATE]` or in the workspace topic. This mirrors the contract that now lives in `KAEL_SYSTEM_PROMPT`.
- **New "Receiving Images" section** documents the v0.12.0 `[SEND_IMAGE path="..." caption="..."]` feature: how to ask for an image, how auto-compression works under the hood, and the strict rule that agents only emit the tag on explicit user request.
- **Configuration section expanded** from a single Telegram-only table to four tables: Common, Telegram, Discord, Slack. All per-platform environment variables are documented.
- **Commands section renamed** from "Telegram Commands" to "Commands" with an explicit note that the command set is identical across all three platforms.
- **Auto-Updates section rewritten** to reflect:
  - The robust `apply_update` flow from v0.12.2: full pip logging, return-code checking, 10-minute timeout, automatic rollback on pip failure.
  - The new startup dependency safety net `bot/_bootstrap.py` that reruns `pip install` when `requirements.txt` changes since the last successful install.
  - The post-update migration framework (`bot/migrations.py`) introduced in v0.12.1.
- **"Workspaces" creation flow** updated: step 1 now says "topic/channel created" with a note clarifying the mapping per platform (forum topic on Telegram, channel on Discord/Slack).
- **Project Structure** section updated:
  - Added the new files introduced in recent releases: `bot/_bootstrap.py`, `bot/media.py`, `bot/migrations.py`.
  - Added the `bot/messaging/` subpackage with `base.py`, `telegram.py`, `discord.py`, `slack.py`.
  - Added `data/migrations.json` (applied migrations tracker).
  - Test count updated from "500+" to "630+".
- **Table of Contents** updated with the new "Receiving Images" and "Platform Migration" entries; "Telegram Commands" renamed to "Commands".

**`ORCHESTRATOR.md`**

- Opening line no longer pins KaelOps to Telegram — it now mentions all three platforms.
- "You live in the **Main** channel" → "You live on the **Command Bridge**".
- Coordination-first contract added right in the intro (mirrors the system prompt and the README).
- Rule #2 ("You post ONLY in Main") updated to "You post ONLY on the Command Bridge".
- New rule #3 explicitly forbidding project-specific execution from the Bridge.
- Workspace creation flow updated: "Create a Telegram topic" → "Create a topic/channel for the workspace (forum topic on Telegram, channel on Discord/Slack)".

**`SCHEDULER.md`**

- Responsibility #7 and the corresponding rule now refer to the Command Bridge instead of "Main channel" and "Telegram notifications".

### What was NOT touched

- `AGENTS.md` — developer-facing engineering notes, already accurate.
- `CHANGELOG.md` — historical records of previous releases stay as-is (they legitimately reference "Main channel" as the pre-Command-Bridge concept).
- `bot/config.py` system prompts — already updated in v0.12.1.
- Install scripts and setup wizard — already platform-agnostic.

## Compatibility

No code, schema, config, or dependency changes. Pure documentation refresh. Drop-in upgrade from v0.12.2.
