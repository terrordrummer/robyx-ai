## What's New

### Non-interactive setup via CLI flags
`setup.py` now accepts every config value as a command-line flag, enabling scripted / headless installs:

```bash
python3 setup.py \
  --platform discord \
  --discord-token "$DISCORD_TOKEN" \
  --anthropic-key "$ANTHROPIC_API_KEY" \
  --owner-id 123456789 \
  --non-interactive
```

Any flag omitted falls back to the interactive prompt, so you can also pre-fill partial config and still walk through the rest. Great for CI, Docker builds, and reproducible deployments.

### Discord setup auto-detection
When configuring Discord, setup now:
- Auto-detects the server the bot has joined (no manual guild ID)
- Creates the forum channel automatically
- Finds the owner ID from the guild membership

All the user has to provide is the bot token — same "one-field setup" experience Telegram has had.

### Detailed setup guidance
Setup prompts and README now include step-by-step "how to obtain" instructions for every parameter across all three platforms (Telegram, Discord, Slack), with explicit pointers to Developer Mode, bot permission scopes, and token locations. New users can get through setup end-to-end without external documentation.

### Uninstall instructions
README documents a clean uninstall path, including warnings about the keep-alive / auto-restart scheduler so users don't fight their own system while tearing down.

## Fixes

- **Scheduler log uses UTC.** `bot/scheduler.py` timestamps are now UTC, matching the rest of the logging stack and avoiding timezone drift when correlating logs across machines.

## Compatibility

No schema or config changes. Drop-in upgrade from v0.10.0 — just pull and restart.
