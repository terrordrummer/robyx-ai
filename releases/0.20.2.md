## Interrupt Any Busy Agent on User Message

User messages now always interrupt a running subprocess instead of
queuing behind the lock. Previously this only worked for continuous-task
topics — now it applies to **all agent types** (interactive, scheduled,
one-shot, continuous). SIGTERM → 5 s grace → SIGKILL, then the new
message is processed immediately.

This is the foundation that later releases (0.20.6+) built on to make
the bot feel responsive regardless of what the agent was doing when
the user spoke.
