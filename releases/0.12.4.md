## What's New

### Python-native reminder engine (`bot/reminders.py`)

Replaces the LLM-based hourly `assistant-check` task for time-sensitive reminders with a
pure-Python scheduler that fires at minute-level precision.

**How it works:**

- Runs every 60 seconds via APScheduler (no Claude Code involved).
- Reads `data/reminders.json` — a structured list of reminder objects.
- For each `status: "pending"` entry whose `fire_at ≤ now`, sends the message directly via
  the Telegram Bot API and marks it `status: "sent"`.
- **Late-firing**: if the service was down and restarts, all past-due reminders are fired
  immediately on the first tick (nothing is silently dropped).
- Each fired reminder records `sent_at` and, when applicable, `late_by_seconds` for
  observability.

**Reminder object format (`data/reminders.json`):**

```json
[
  {
    "id": "r-a3f7b2c1",
    "message": "⏰ Riunione con il team tra 1 ora",
    "fire_at": "2026-04-09T09:00:00+02:00",
    "thread_id": 903,
    "created_at": "2026-04-08T10:00:00+02:00",
    "status": "pending"
  }
]
```

`fire_at` is ISO 8601 with explicit timezone offset. Timezone-naive datetimes are treated
as UTC for safety.

### Intelligent pre-notification in the assistant agent

`agents/assistant.md` updated with a pre-notification table: the assistant agent decides
autonomously how early to remind based on event type, unless the user specifies otherwise:

| Event type | Default lead time |
|---|---|
| Important appointment (meeting, flight) | Day before at 09:00 + 1 hour before |
| Deadline (payment, submission) | 2 days before + day-of at 09:00 |
| Generic reminder | Exact time requested |
| Quick call / task | 15 minutes before |
| Recurring event (e.g. every Thursday) | Day before at 20:00 |

Multiple reminder objects are written per event when pre-notification applies.

### `assistant-check` task demoted

The `assistant-check` LLM task is now `every-6h` (was `hourly`) and uses `haiku` (was
`sonnet`). It handles proactive to-do nudges only — time-critical reminders are fully
handled by the Python engine.

### AI timeout raised to 2 hours

`AI_TIMEOUT` in `bot/config.py` raised from 600 s (10 min) to 7200 s (2 h) to
accommodate long-running R&D and build tasks (e.g. Zeus Focus Stacking engine benchmarks
that take well over 10 minutes).

## Compatibility

No schema changes beyond the new `data/reminders.json` file (auto-created as `[]` on
first run if missing). Drop-in upgrade from v0.12.3 — pull and restart.
