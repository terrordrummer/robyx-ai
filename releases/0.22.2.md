# Robyx 0.22.2

Patch release: **Pass 2 security hardening** — the SEC-only slice of the
full Pass 2 code review (spec 002). Stability / UX / Natural-Interaction
slices carry over to a subsequent cycle unchanged.

## User-visible symptom (before this release)

No functional change for end users under normal operation. This release
tightens trust boundaries at ingestion points and subprocess boundaries
so that:

- A voice-message flood on Telegram can no longer fill operator disk.
- A prompt-injected orchestrator can no longer write markdown outside
  the workspace `data/agents/` directory via `[COLLAB_ANNOUNCE name="…"]`.
- A malicious `setup.py` in a transitive pip dep can no longer read bot
  tokens or AI provider keys from the environment when the startup
  safety net re-runs `pip install` after a `requirements.txt` change.
- Closing a collaborative workspace now also revokes the operator's
  ability to drag the bot into new Telegram groups.
- OpenAI Whisper transcription rejects files exceeding the 25 MB cap
  up-front instead of burning bandwidth on a doomed round trip.

### New fixes in this release (beyond what was already in 0.22.0 / 0.22.1)

Six new Pass 2 SEC fixes landed in this cycle (commits on `002-full-code-review`):

- **P2-80** — `is_authorised_adder` now scopes to non-closed workspaces so `/close` is a real trust-revocation signal (`bot/authorization.py`).
- **P2-81** — shared `validate_collab_name()` helper gates every AI-emitted collaborative workspace name against path traversal, wired into both `_handle_collab_announce` and `CollabStore.create_pending` (`bot/collaborative.py`, `bot/handlers.py`).
- **P2-82** — `TelegramPlatform.download_voice` enforces a 25 MB size cap with pre-download + post-download checks and unconditional temp-file cleanup (`bot/messaging/telegram.py`).
- **P2-83** — `voice.transcribe_voice` refuses oversize audio up-front via a new `voice_too_large` i18n string; handlers.py wraps the transcribe call in `try/finally` so the `.ogg` tempfile is cleaned on every exit path including cancellation (`bot/voice.py`, `bot/handlers.py`, `bot/i18n.py`).
- **P2-84** — `resolve_db_path` / `get_memory_dir` reject specialist `agent_name` values containing path separators, `.` / `..`, or control characters (`bot/memory_store.py`, `bot/memory.py`).
- **P2-86** — `_bootstrap.ensure_dependencies` spawns pip with a scrubbed environment mirroring `bot/updater.py::_CHILD_ENV_SCRUB` (platform tokens + AI provider keys stripped) so startup-path reinstalls do not leak secrets to transitive deps (`bot/_bootstrap.py`).

Accompanying audit closures with no code change but updated tests /
documentation: **T068** (bot.py lifecycle), **T069** (scheduler queue /
work_dir / retry), **T073** (messaging ABC uniformity), **T079**
(_bootstrap + process clean).

## Upgrade notes

- Operators on 0.22.1 upgrade to 0.22.2 via the built-in auto-updater
  or `git pull` + `systemctl restart robyx`. No operator action is
  required.
- The only new user-facing string is `voice_too_large` ("Voice message
  is too large to transcribe (%d MB, cap %d MB). Please record a
  shorter clip."). Surfaces only when a voice message exceeds the 25 MB
  Whisper ceiling.
- Existing pending / setup / active collaborative workspaces with
  legacy names that violate the new `validate_collab_name` pattern
  remain readable (the validator only gates **new** Flow-A announces
  and new `create_pending` calls; existing roundtrips are unaffected).
