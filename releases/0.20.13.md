## Test-suite stabilization + two real production bugs in the update path

44 tests had been failing on `main` since v0.20.6's "convert all blocking
subprocess.run calls to async" sweep. Treating regressions as "pre-existing
failures" was masking actual production bugs — including two in the
self-update flow that surface every time someone runs `/checkupdate` or
`/doupdate`.

### Production bugs fixed

1. **`bot/updater.py`** — `get_pending_update()` was declared `def`
   (synchronous) but its body called the async `fetch_remote_tags()` and
   `_get_release_notes_for()` *without `await`*. The function silently
   returned a dict whose values were unawaited coroutines. Any caller
   that did `pending["version"]` got a `TypeError` ("coroutine object is
   not subscriptable"). Now properly `async`.

2. **`bot/handlers.py`** — `cmd_checkupdate` and `cmd_doupdate` wrapped
   the async `check_for_updates()` / `get_pending_update()` in
   `asyncio.get_event_loop().run_in_executor(None, ...)`. That executes
   the coroutine **factory** in a worker thread, which returns an
   unawaited coroutine; awaiting the executor future then yields the
   coroutine itself instead of the dict the rest of the handler expects.
   Replaced both with direct `await`s. The `/checkupdate` and
   `/doupdate` commands actually work again.

### Test-infra changes (no behaviour change)

- **`tests/test_process.py`** (14 tests) — split into async + sync
  classes, mirroring the `*_sync` / async pair v0.20.6 introduced.
- **`tests/test_scheduler.py::TestCheckLock`** (5 tests) — converted
  to `@pytest.mark.asyncio`, `is_ai_process` mocked with `AsyncMock`.
- **`tests/test_updater.py`** (20 tests) — `_git`, `fetch_remote_tags`,
  `_get_release_notes_for`, `check_for_updates`, `get_pending_update`
  tests all rewritten to await the async API with `AsyncMock` patches.
- **`tests/test_handlers.py::TestCmdCheckUpdate`** (5 tests) — pass
  automatically thanks to the production fix above.
- **`tests/test_bot.py::TestEnsureSingleInstance::test_pid_reused_by_non_python`**
  — was patching the wrong helper names (`is_bot_process` instead of
  `is_bot_process_sync`); fixed.
