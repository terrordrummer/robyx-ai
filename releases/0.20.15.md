## Documentation split: focused docs, lighter README

The README had grown to 809 lines covering everything from "what is Robyx"
to "how the auto-update flow works internally". This release splits it
into a concise landing page plus topical docs.

### New layout

```
README.md (300 lines)
├── Why Robyx
├── Documentation index table  ← jump to any topic
├── Quick Start (3 steps)
└── Project Structure
docs/
├── architecture.md          # How it works, three roles, workspaces
├── scheduler.md             # Reminders, one-shot, periodic, continuous
├── memory.md                # Active + archive memory tiers
├── team.md                  # How to grow your fleet
├── configuration.md         # Every env var, per-platform settings
├── commands.md              # Slash command reference
├── backends.md              # Claude/Codex/OpenCode + permissions + models.yaml
├── media.md                 # Voice + images
└── updates.md               # Auto-updates + migrations + service mgmt
```

Each docs file links back to the README at top and bottom; the README
has a documentation index table mapping each topic to its file.

### Files touched

- `README.md` — rewritten as a landing page; full content preserved
  verbatim in `docs/`
- `docs/*.md` — 9 new files
- `tests/test_docs_contract.py` — updated to look at README + relevant
  docs files concatenated, since the cross-platform contract is now
  spread across multiple files
- `bot/migrations/v0_20_15.py` (no-op, keeps the chain continuous)
- `VERSION`, `CHANGELOG.md`, `releases/0.20.15.md`

### No migration needed

Pure documentation reorganization — no code, schema, or behaviour
changes. The bootstrap migration `v0_20_15.py` is a no-op.
