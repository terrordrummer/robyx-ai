## What's New

### Multi-platform abstraction layer
- New `bot/messaging/` package with abstract `Platform` interface and `PlatformMessage` dataclass
- `TelegramPlatform` adapter implements the full interface for Telegram
- All handler, AI invocation, and topic management code now uses the Platform abstraction instead of Telegram objects directly
- New `KAELOPS_PLATFORM` config variable (defaults to `telegram`)
- Foundation for Discord and Slack adapters in future releases

### Architecture changes
- `handlers.py`: all handlers now receive `(platform, msg, msg_ref)` instead of Telegram's `(update, context)`
- `ai_invoke.py`: `bot` parameter replaced with `platform` throughout; typing indicators and status messages use platform methods
- `topics.py`: removed Telegram-specific HTTP calls; `create_workspace()`, `close_workspace()`, `create_specialist()` now accept a `platform` parameter
- `bot.py`: creates platform instance from config, wraps Telegram events into `PlatformMessage`
- No Telegram imports outside of `bot/messaging/telegram.py` and `bot/bot.py` (the entry point bridge)

### Platform migration
- Tell Kael "switch to Discord" or "passa a Slack" to migrate between platforms
- Kael guides you through the one manual step (creating a bot token), updates config, and restarts
- All workspaces, agents, scheduled tasks, and memory are preserved during migration

### Test suite
- 516 tests passing
- All tests refactored to use `mock_platform` fixture instead of Telegram-specific mocks
- `PlatformMessage` used throughout tests instead of mocked Telegram `Update` objects
