## Fixes

### Topic routing no longer migrates conversations silently
If you asked something in a forum topic / channel that wasn't bound to any workspace agent, KaelOps would silently route the message to Kael and redirect the response to the main channel — the typing indicator would disappear from your topic and reappear in `#general`, which looked like the bot had crashed mid-reply.

The router now has three explicit branches:

1. **Topic is mapped to an agent** → the agent responds in the topic as before.
2. **Message comes from the platform's main destination** → Kael (or the focused/mentioned agent) responds there.
3. **Unmapped topic** → KaelOps replies *in the same topic* with a clear hint explaining that the topic has no agent, and does **not** invoke any AI. No more silent migrations.

A warning log entry is written whenever branch 3 triggers, so you can see it in `data/runtime/bot.log`.

### Discord and Slack main-channel routing
The legacy `MAIN_THREAD_ID = 1` sentinel was a Telegram-specific idea that leaked into the other adapters:

- **Discord**: Kael's "main" was being resolved as `thread_id=1`, causing `Client.fetch_channel(1)` to fail. Kael could not respond in the control channel.
- **Slack**: `thread_ts="1"` is not a valid Slack timestamp, so top-level replies in the control-room channel broke the same way.

The fix introduces a `Platform.is_main_thread(chat_id, thread_id)` abstract method with the correct per-platform semantics:

- **Telegram**: `thread_id is None` (the General topic of a forum supergroup).
- **Discord**: `thread_id == control_channel_id`.
- **Slack**: top-level messages in the control-room channel (`thread_id is None and chat_id == control_channel`).

The old redirect logic is gone entirely.

### Robust error delivery
- `_process_and_send` now routes error messages through a new `_safe_send` helper that tries markdown first and falls back to plain text on failure. You will always see what went wrong — no more "Kael vanished" when an error string happened to contain markdown specials.
- `_send_response` guards against empty responses after pattern stripping, substituting `[Empty response]` instead of sending just the agent tag.
- `_handle_workspace_commands` wraps each `create_workspace` / `create_specialist` in its own try/except so a single failure no longer aborts an entire batch of multi-creation.
- `invoke_ai` now awaits the keep-alive task after cancelling it in `finally`, eliminating orphaned typing requests and silencing "coroutine was never awaited" warnings.

## Tests

Suite grew from 567 to 576 tests. New coverage:

- Unmapped topic → reply in-place, no AI invoked, correct hint string.
- Main-thread preservation (no redirect).
- Markdown-to-plain fallback in `_process_and_send`.
- `is_main_thread` behaviour on Discord and Slack adapters.

## Compatibility

No schema, config, or state-file changes. Drop-in upgrade from v0.11.0 — just pull and restart.
