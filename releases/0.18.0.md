## System monitor on-demand only; installer disables push

### System monitor removed from scheduler

`data/tasks.md` no longer schedules `system-monitor` every six hours.
The agent file (`agents/system-monitor.md`) is retained — any workspace
agent or the orchestrator can still invoke it on demand — but the
periodic health snapshot is gone. The every-6h slot added no signal
that wasn't already visible through normal bot operation and generated
noise in the topic.

### Installer disables remote push (read-only install)

`install/install-mac.sh` now runs:

```bash
git remote set-url --push origin no_push
```

immediately after the setup wizard. This sets the push URL of `origin`
to the literal string `no_push`, which is not a valid remote. Any
subsequent `git push` from the install directory fails with a clear
error; `git fetch` and `git pull` are unaffected, so the auto-updater
(`bot/updater.py`) continues to work normally.

The motivation: Claude Code, when invoked inside the install directory,
can see the `.git` folder and — if it decides to commit a local
change — has a live push path to the public repository. Setting the
push URL to a sentinel value closes that path without removing version
history or breaking pull-based updates.

The guard runs only when `origin` exists (fresh clones always have it;
manually assembled installs without a remote are left untouched).

## Compatibility

- **Non-breaking.** No data migrations, no schema changes, no new
  dependencies. Existing installs can apply the patch by pulling and
  running `git remote set-url --push origin no_push` once manually,
  or by re-running `install/install-mac.sh`.
- **Auto-updater unaffected.** `bot/updater.py` uses `git fetch` +
  `git reset --hard`, neither of which touches the push URL.
