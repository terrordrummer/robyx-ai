## What's New

### Real-time streaming & progress updates
- Claude CLI output is now streamed via `stream-json` format instead of waiting for the full response
- Agents can emit `[STATUS ...]` markers that are relayed to Telegram instantly as the AI works
- Multi-workspace creation shows step-by-step progress (`1/3 — Creating workspace X...`)

### Single-instance protection
- Bot writes a PID file (`data/bot.pid`) at startup
- If another instance is already running, the new one exits immediately with a clear error
- Prevents the duplicate-bot 409 Conflict issue that could occur from manual launches

### Agent memory system
- Persistent two-tier memory: **active** (loaded into context) + **archive** (queryable on demand)
- Projects with existing Claude Code memory (`.claude/`, `CLAUDE.md`) are left untouched — native memory works as-is
- Projects without memory get `.kaelops/memory/` with active + archive
- Specialist and orchestrator memory lives centrally at `data/memory/{name}/`
- Agents update memory continuously — not at session boundaries
- Active memory budget: ~5000 words; excess is archived by quarter

### Multi-language support
- All agents (Kael, workspaces, specialists) now reply in the same language the user writes in

### Voice transcription feedback
- Voice messages now show the transcription as a reply before the agent processes the content

### Cross-platform compatibility
- New `process.py` module: cross-platform PID checking (tasklist on Windows, ps on Unix)
- Windows: pip path, service restart, lock file deletion all work natively
- Windows installer checks for Administrator privileges
- SIGTERM signal handler conditional on Unix only
- Linux: verified systemd integration, no changes needed
