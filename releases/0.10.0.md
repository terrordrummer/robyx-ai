## What's New

### Three messaging platforms
KaelOps now supports **Telegram**, **Discord**, and **Slack** with full feature parity:

| Feature | Telegram | Discord | Slack |
|---------|----------|---------|-------|
| Text messages | Markdown | Markdown | mrkdwn |
| Voice → Whisper | .ogg | .ogg (attachment) | .webm (audio clip) |
| Workspaces | Forum topics | Forum channel threads | Dedicated channels |
| Typing indicator | Native | Native | No-op (Slack limitation) |
| Commands | /command | /command (prefix) | /command (prefix) |
| Setup automation | Auto-detect chat ID | Auto-create server | Socket Mode (no public URL) |

Choose your platform during setup:
```
$ python3 setup.py

--- Step 0: Messaging Platform ---
  [1] Telegram
  [2] Discord
  [3] Slack
```

### Platform migration via chat
Switch platforms at any time by telling Kael:
> "Passa a Discord" / "Switch to Slack" / "Migrate to Telegram"

Kael guides you through the one manual step (bot token), updates config, and restarts. All workspaces, agents, scheduled tasks, and memory are preserved.

### Discord adapter
- Full `DiscordPlatform` implementation using `discord.py`
- Forum channels for workspaces, text channels for control room
- Voice message support via attachment download
- Auto-create guild, channels, and invite link during setup
- Background scheduler/update loops via asyncio

### Slack adapter
- Full `SlackPlatform` implementation using `slack-bolt` + `slack-sdk`
- Socket Mode — no public URL or webhook endpoint needed
- Audio clip transcription via Whisper
- Channels for workspaces, threads for conversations
- Bot token (`xoxb-`) + App token (`xapp-`) authentication

### Platform abstraction layer
- `bot/messaging/base.py` — `Platform` ABC + `PlatformMessage` dataclass
- All core logic (handlers, AI invoke, scheduler, memory) is fully platform-agnostic
- Adding a new platform = one file implementing the Platform interface

### Test suite
- **566 tests** (up from 516)
- 25 Discord platform tests + 25 Slack platform tests
- All tests use `mock_platform` fixture — platform-agnostic by design
