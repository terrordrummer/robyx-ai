## Suppress False Error on Agent Interruption

Interrupted agents no longer display `AI Error: unknown` to the user.

An `interrupted` flag was added to `Agent` that is set during
`interrupt()` and checked in the invoke path. Terminated subprocesses
now return `None` instead of an error string, and handlers silently
skip sending — so a clean interrupt followed by a new user message
produces no spurious error bubble in the chat.

Follow-up to v0.20.2, which made "interrupt on any user message" the
default for every agent type.
