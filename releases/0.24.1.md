# Robyx 0.24.1

Documentation patch: purge residual references to the pre-0.23.0
"dedicated workspace topic per continuous task" model from the files
that workspace agents read at setup time.

## Why this patch

Spec 005 (v0.23.0) unified continuous-task I/O onto the parent workspace
chat — no dedicated sub-topic is created anymore; step reports flow into
the workspace chat prefixed with `🔄 [<name>]`. The runtime behaviour
has been correct since 0.23.0, but a handful of prompt templates and
docs still described the old model, leading workspace agents to promise
the user "a dedicated topic" during `/loop` setup — a promise the
runtime then silently did not honour.

## What changed

### Documentation

- `templates/CONTINUOUS_SETUP.md` — the final paragraph of the macro
  emission section no longer claims the system will "create a dedicated
  workspace topic". It now describes the actual lifecycle: git branch
  in `work_dir`, queue registration, step reports back into the parent
  workspace chat with the `🔄 [<slug>]` prefix.
- `docs/architecture.md` — the "agentic loop" paragraph (workspace-agent
  section) reflects the unified-chat model: dedicated git branch +
  per-task state, reports routed to the workspace chat with the 🔄
  marker.
- `templates/prompt_workspace_agent.md` — two stale occurrences fixed:
  the "when NOT to use" bullet on overhead cost, and the conversational
  suggestion template ("I can set it up as a continuous task with …").
  Both now describe branch + state file, not topic.
- `templates/prompt_focused_agent.md` — same suggestion template
  realigned.

### No functional change

- No schema change, no state migration logic. `bot/migrations/v0_24_1.py`
  is a no-op and exists only to keep the version chain continuous.
- No API, prompt macro, or behaviour change at runtime.

## Files not touched

Historical and spec artifacts that describe the *old* sub-topic model by
design are intentionally left unchanged:

- `CHANGELOG.md` entries for versions ≤ 0.22.x (historical record).
- `releases/0.20.0.md`, `releases/0.23.0.md` (release-time snapshots).
- `specs/004-*`, `specs/005-*` (planning artifacts; spec 005 explicitly
  documents the model it *replaces*).
- `bot/migrations/v0_23_0.py`, `tests/test_migration_v0_23_0.py` —
  the migration code that operates on legacy sub-topics.
- `bot/topics.py` docstrings that reference "spec 005 removed the
  dedicated sub-topic model" (these document the decision, not a
  current capability).

## Upgrade notes

- Operators on 0.24.0 upgrade to 0.24.1 via the built-in auto-updater
  or `git pull` + `systemctl restart robyx`. No operator action is
  required.
- Workspace agents reload `templates/prompt_workspace_agent.md` on the
  next turn; from that point the `/loop` setup prose matches the actual
  lifecycle the user observes.
