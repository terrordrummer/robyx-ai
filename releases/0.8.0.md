## What's New

### Agent persona enforcement
- Kael and all agents now maintain their identity consistently — they never reveal themselves as Claude, an AI model, or an LLM
- Kael no longer says "I don't have access to Telegram" — he responds as the bot he is

### Live configuration via chat
- Tell Kael an API key or config value and he'll update `.env` and restart the service automatically
- New `[RESTART]` command pattern: agents can trigger a graceful service restart after config changes
- No need to SSH into the server or edit files manually for routine config updates

### Graceful voice handling without API key
- Voice handler is now always registered — if `OPENAI_API_KEY` is missing, the bot replies with a clear, actionable message instead of silently ignoring the voice
- Error messages are user-friendly and in Italian (matching the user's language)
- `transcribe_voice()` now returns a `(text, error)` tuple instead of using fragile string-prefix conventions

### Auto-update for safe releases
- Non-breaking, compatible updates are now applied automatically when detected (hourly check)
- Breaking or incompatible updates still notify and require manual `/doupdate`
- Notification messages show release notes before auto-applying

### Simplified setup wizard
- Setup auto-detects Chat ID and Owner ID when the bot is added to a Telegram group
- No more manual copy-pasting of negative chat IDs or user IDs
- Uses Telegram long polling on `my_chat_member` events for instant detection
- Ctrl+C falls back to manual entry if auto-detection doesn't work
- Voice step simplified: "enter key or skip" instead of yes/no + key + language

### Whisper auto language detection
- Removed hardcoded `WHISPER_LANGUAGE=en` — Whisper now auto-detects the spoken language
- Italian voice messages are transcribed in Italian, English in English, etc.
- Removed `WHISPER_LANGUAGE` config variable entirely

### Documentation
- ORCHESTRATOR.md updated with configuration management section and identity rule
- README.md updated with live config, voice graceful-degradation, and auto-update docs

### Bug fixes
- Increased subprocess stdout buffer to 1MB for large AI responses

### Test suite
- 526 tests (up from 515)
- New tests for `[RESTART]` pattern, voice-not-available early return, auto-update flow, and updated voice tuple API
