# Robyx 0.22.1

Patch release: close the continuous-task macro leak.

## User-visible symptom (before this release)

When a user asked an agent to set up a continuous task, the reply
occasionally contained the raw protocol block:

```
[CREATE_CONTINUOUS name="..." work_dir="..."]
[CONTINUOUS_PROGRAM]
{ ... full JSON payload ... }
[/CONTINUOUS_PROGRAM]
```

No topic, no branch, no scheduled step were created.

## Fix

`bot/continuous_macro.py` consolidates detection, stripping, validation,
and side-effect dispatch for the continuous-task macro. It runs at a
single point in `_process_and_send` before any other marker handler, so:

- both the orchestrator and workspace-agent executive paths are covered
  (the routing gap the leak depended on is gone);
- stripping is unconditional — any partial token is scrubbed, whether or
  not its partner matched;
- detection tolerates realistic LLM variations: case-insensitive tags,
  curly quotes, extra whitespace, triple-backtick code fences, and
  multiple macros per response;
- every rejection path renders a short prose error from `i18n`
  (no raw tokens, no JSON);
- scheduled subprocess output is also scrubbed defensively (it never
  dispatches a new task — it only strips).

## No migration required

The `data/continuous/<name>/state.json` schema is unchanged. Tasks created
before the upgrade run unmodified.

## Tests

`tests/test_continuous_macro.py` adds 53 tests (pure + fixture-driven +
async side-effect dispatch). `tests/test_handlers.py` adds three new
integration cases under `TestContinuousMacroInterception`, and
`tests/test_scheduled_delivery.py` adds one regression case under
`TestContinuousMacroScrubbing`. The full suite remains green (1450
passed, 1 skipped).

## Rollback

`git revert <fix-commit> && systemctl restart robyx` — no state cleanup
required.
