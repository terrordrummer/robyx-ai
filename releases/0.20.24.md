## Continuous task recognition and `/loop` trigger in agent prompts

Workspace agents had the technical ability to create continuous tasks
since v0.20.20, but lacked the judgment to know *when* to propose them.
The result: faced with iterative long-running work, agents would attempt
to execute everything in a single session instead of setting up the
structured loop mechanism.

### Changed -- workspace agent prompt

The "Continuous Tasks" section has been rewritten with three key additions:

1. **Two activation modes**: agents now respond to both an explicit
   `/loop` command (interpreted in context -- setup trigger vs.
   discussion about the mechanism) and conversational deduction (the
   agent proactively suggests a continuous task when it recognizes
   signals like "iterative R&D", "optimize until metric X", "try
   different strategies one at a time").

2. **"When in doubt, suggest it" rule**: the cost of an unnecessary
   suggestion is one message; the cost of running a long task inline is
   a failed or incomplete execution with no structured history.

3. **Stronger setup interview**: agents are instructed to challenge vague
   criteria ("ottimizza" is not a stopping criterion; "riduci il tempo
   sotto 500ms" is).

### Changed -- focused agent prompt

Added a "Continuous Tasks (agentic loop)" section so agents operating in
focus mode (direct user chat) also recognize iterative work patterns and
can suggest/create continuous tasks. Previously focus mode had zero
awareness of this mechanism.

### Migration

None. ``bot/migrations/v0_20_24.py`` is a no-op. Prompt-only change.
