## Pre-update data backup + post-update smoke test

The migration framework added in v0.20.12 makes ordering and tracking
safe, but until now an update could still leave the bot in a worse
state than it started in: a botched migration could mutate `data/`
before the failure was detected, and a green `pip install` could still
hide a broken runtime that only crashes at import time.

This release closes both gaps with two complementary safety guards
wired into `apply_update`.

### 1. Pre-update tar snapshot of `data/`

Right after `git stash` and before `git pull`, `apply_update` now calls
`_snapshot_data_dir()` to write a versioned tar+gz of every file under
`DATA_DIR` (excluding the `backups/` subdir itself):

```
data/backups/pre-update-0.20.13-to-0.20.14-20260415T101234Z.tar.gz
```

Retention keeps the most recent 3 snapshots; older ones are pruned by
mtime. Snapshot failure is logged at WARNING but does **not** block the
update — running unprotected is still better than refusing to update
because the disk is full.

Every rollback path in `apply_update` now restores from this snapshot:
- migration step non-zero exit
- migration step timeout
- post-update smoke test failure (new — see below)
- catastrophic exception in the outer try

So a partially-mutated `data/` directory cannot survive a failed update.

### 2. Post-update import smoke test

After `pip install` succeeds and *before* the success state is
recorded, `_post_update_smoke_test()` spawns

```
<venv>/bin/python -c "import bot.bot"
```

with a 60s timeout. Pip exit 0 isn't enough — a successful resolve can
still leave the runtime broken (transitive dependency conflict surfacing
at import time, syntax error from a partial commit, missing module
constant referenced by a freshly-imported migration). Catching that here
lets the caller roll back instead of restarting straight into a broken
bot.

On smoke test failure:
1. `git checkout v<previous>` — restore code.
2. `_restore_data_dir(snapshot)` — restore data.
3. `git stash pop` — restore the user's local changes (if any were
   stashed in step 1 of the update flow).
4. Return `(False, "Smoke test failed: ...")` with a tail of the import
   error so the chat notification stays readable.
