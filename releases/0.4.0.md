## Changes
- Update commands now show live progress (message edits in-place instead of typing silence)
- `/checkupdate` runs git fetch in a background executor to avoid blocking
- `/doupdate` checks for busy agents and running scheduled tasks before updating
- `/doupdate force` to override busy checks when needed
- Boot notification: "KaelOps vX.Y.Z started." sent to General on every service start
- Robust error handling in `_process_and_send` — errors always produce a user-visible message
