## KaelOps is now Robyx

KaelOps has been rebranded to **Robyx** and moved to a new repository:

```
https://github.com/terrordrummer/robyx-ai
```

development continues on `robyx-ai`.

## How to migrate

### Option A: Migration script (recommended for existing installs)

```bash
# 1. Clone the new repo
git clone https://github.com/terrordrummer/robyx-ai.git
cd robyx-ai

# 2. Run the migration script, pointing to your old install
./migrate-from-kaelops.sh /path/to/your/kael-ops

# That's it. The script:
# - Stops the old com.kaelops.bot service
# - Copies data/ and .env to the new location
# - Renames KAELOPS_* env vars to ROBYX_*
# - Installs the new com.robyx.bot service
# - On first boot, a migration renames kael→robyx in state.json,
#   moves memory directories, and resets AI sessions
```

### Option B: Fresh install

```bash
git clone https://github.com/terrordrummer/robyx-ai.git
cd robyx-ai
./install/install-mac.sh    # or install-linux.sh / install-windows.ps1
```

### Option C: In-place upgrade (advanced)

If you want to keep using the same directory:

```bash
cd /path/to/your/kael-ops
git remote set-url origin https://github.com/terrordrummer/robyx-ai.git
git fetch origin
git reset --hard origin/main
./install/install-mac.sh    # reinstall with new service name
```

Note: the old `com.kaelops.bot` launchd service must be manually unloaded
before installing the new `com.robyx.bot`:

```bash
launchctl unload ~/Library/LaunchAgents/com.kaelops.bot.plist
rm ~/Library/LaunchAgents/com.kaelops.bot.plist
```

## Backward compatibility

- **`.env` files with `KAELOPS_*` vars still work.** The new code reads
  `ROBYX_*` first, falls back to `KAELOPS_*`. The boot-time migration
  renames them automatically.
- **`data/state.json` with agent name `kael` is auto-migrated.** The
  `0.19.0-rename-kaelops-to-robyx` migration renames it to `robyx` on
  first boot.
- **Workspace memory directories** (`.kaelops/memory/`) are moved to
  `.robyx/memory/` automatically.
- **No data is lost.** The migration is additive — it renames, never deletes.
