## Consistent stream-retry across all backends

v0.20.9 introduced auto-retry on transient Claude Code stream errors, but the
`STREAM_RETRYABLE_KEYWORDS` list only covered Claude-specific phrases. The
retry *logic* (in `_invoke_ai_locked`) is already shared by every backend, so
the one thing still missing was keyword coverage for Codex and OpenCode.

### Changed

- **`bot/ai_invoke.py`** — `STREAM_RETRYABLE_KEYWORDS` now covers:
  - Claude Code: `stream idle timeout`, `partial response received`
  - Node.js (Codex): `socket hang up`, `fetch failed`, `network timeout`,
    `econnreset`, `etimedout`
  - Go / OS-level (OpenCode + generic): `connection reset`,
    `connection refused`, `broken pipe`, `context deadline exceeded`,
    `unexpected eof`
  Any match on stderr, stdout, or the parsed result payload triggers a
  session regeneration + exponential-backoff retry, up to `MAX_AI_RETRIES`.

### Added

- **`tests/test_ai_invoke.py`** — `test_stream_retryable_works_for_all_backends`
  parametrized over `claude_backend`, `codex_backend`, `opencode_backend`.
  Exercises the non-streaming failure → retry → recovery path with a
  realistic transient-error payload (`socket hang up`) and asserts that all
  three backends retry once and return the recovered response.

### Files touched

- `bot/ai_invoke.py`
- `tests/test_ai_invoke.py`

### No migration needed

Pure logic additions; no config, schema, or queue format changes.
