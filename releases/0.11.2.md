## Fixes

### Endless "typing…" on session-ID collision
Observed: after upgrading to v0.11.1 a real user sent a message to a workspace topic and saw the typing indicator stuck forever, no reply ever arrived. Root cause turned out to be a chain of three bugs, all fixed here:

1. **Placeholder session IDs in state files**: some agents carried deterministic sequential ids like `00000000-0000-0000-0000-000000000003` (leaked in from earlier state files / manual edits). The Claude CLI remembers every session id it has ever seen and rejects the second invocation with `Error: Session ID ... is already in use`.

2. **Retry loop reused the same id**: `_invoke_ai_locked` already had a retry branch for "already in use" errors, but it recursed with the *same* `agent.session_id`. It could never succeed — and because the retry flipped `agent.session_started = True`, the second attempt switched to `claude --resume <bad_id>`, which on Claude Code hangs instead of failing fast.

3. **Unreasonable AI timeout**: `AI_TIMEOUT` defaulted to **3600 seconds (1 hour)**. Combined with the hang above, the keepalive loop would send a `sendChatAction` every 5 seconds for a full hour before anything gave up.

### What changed

- **`bot/agents.py`**: added `_is_placeholder_session_id` detector and load-time sanitisation in `AgentManager._load_state`. Any session id that is empty, matches the `00000000-0000-0000-0000-*` family, or does not parse as a valid UUID is regenerated as a fresh `uuid.uuid4()`, session state is reset (`session_started=False`, `message_count=0`) and the sanitised state is persisted. This heals existing broken state files on the next restart.
- **`bot/ai_invoke.py`**: the "already in use" retry branch now generates a new UUID via `uuid.uuid4()`, resets `session_started`/`message_count`, persists via `manager.save_state()`, and only then recurses. The retry is guaranteed to produce a usable session.
- **`bot/config.py`**: `AI_TIMEOUT` lowered from `3600` to `600` (10 minutes). Long enough for real tool-heavy Claude runs, short enough to surface hangs quickly.

### Tests

Suite grew from 576 to 581 tests. New coverage:

- `TestPlaceholderSessionIdSanitisation`: sequential placeholders, non-UUID strings, valid UUID preserved, sanitised state persisted.
- `TestIsPlaceholderSessionId`: direct unit tests for the detector.
- `test_session_collision_regenerates_session_id`: guarantees the retry path produces a different, valid UUID.

## Operator note — upgrading from 0.11.1

If you already have a running 0.11.1 and notice a stuck typing indicator on some topic, the recovery sequence is:

1. `kill -TERM <bot_pid>` (or `launchctl kickstart -k gui/<uid>/com.kaelops.bot` on macOS). The supervisor will restart the bot.
2. The new 0.11.2 code auto-sanitises placeholder session ids at load time — no manual state editing is required.

Drop-in upgrade from v0.11.1.
